# 系统架构说明

## 1. 目标

用户输入原话后，系统自动识别场景、关系和语气，生成三档沟通优化版本，并支持语音播放与历史追踪。

## 2. 分层设计

- 小程序层：负责输入、展示、播放、历史浏览
- API 层：负责鉴权、参数校验、业务编排
- AI 适配层：统一封装 ASR / LLM / TTS 供应商
- 数据层：MySQL 存核心业务数据，Redis 做缓存和限流，对象存储保存音频

## 3. 核心链路

1. 用户输入文本或上传语音
2. 后端在语音场景下先做 ASR
3. LLM 做场景识别、关系识别、目标识别和三档表达改写
4. 结果落库，并可触发 TTS 生成播放音频
5. 小程序展示结果、支持复制、收藏、反馈、历史查看

## 4. 可替换模型设计

- `LlmProvider`：统一封装 OpenAI / GLM / 本地模型
- `AsrProvider`：统一封装 Whisper / 腾讯云 ASR / 本地 ASR
- `TtsProvider`：统一封装云 TTS / 本地 TTS

业务层不直接依赖具体供应商，只依赖 Provider 接口。
